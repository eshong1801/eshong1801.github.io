BearQueue

Production ticketing platform built for the UC Berkeley Mechanical Engineering department — React frontend, Java backend, AWS-hosted, serving 500+ students.

Overview

BearQueue (internally known as the Ticketing Management System / TMS) is a full-stack web application the UC Berkeley Mechanical Engineering department uses to manage student workflows — queues, office hours, class operations, and activity logging. Live at www.bearqueue.com.

Unlike most school projects, this one had real users from day one and continues to iterate as the department's needs evolve.

What I Built

  • Led the TMS engineering group, coordinating frontend, backend, and infrastructure work across a 150-student initial deployment that grew to 500+ active users.
  • Built the application with a React frontend and a Java backend, designed for the operational realities of the Berkeley ME department rather than a generic queue tool.
  • Deployed to production on Amazon Web Services with MongoDB as the persistence layer and Google Sign-In for user authentication.
  • Designed the system to log every administrative activity — providing the department a durable audit trail for student-support workflows.
  • Currently in active production at www.bearqueue.com, used routinely by ME faculty and staff.

Why It Matters

Most portfolio projects never see a user. This one runs in production, gets used by real students every week, and has to keep working. That's a different engineering discipline than a homework assignment — uptime matters, edge cases bite, and feature requests come from people whose workflows you can actually break.
