Custom 3-Stage RISC-V Processor

A pipelined 3-stage RISC-V CPU implemented in Verilog and synthesized to a Xilinx FPGA — including the M-extension, full hazard handling, and a UART peripheral for off-chip I/O.

Overview

This project implements a 3-stage pipeline (Fetch, Execute, Writeback) in Verilog and synthesizes it onto a Xilinx FPGA development board. The result is real silicon (well, real LUTs) executing RV32IM programs that I write, assemble, and load over UART.

Beyond the textbook pipeline, the design needed to actually handle the messy parts: structural and data hazards, control flow disruption from branches, and the multi-cycle behavior of the M-extension's divide unit.

What I Built

  • Full 3-stage pipeline (IF / EX / WB) with per-stage pipeline registers and a clean module decomposition.
  • Hazard detection unit and WB→EX forwarding paths, eliminating the common RAW stalls without flushing.
  • RISC-V M-extension support: mul, mulh, div, divu, rem, remu — including the multi-cycle iterative divider with pipeline stalling.
  • Memory-mapped UART peripheral allowing the CPU to communicate with a host PC for debugging and program I/O.
  • Custom RV32 assembly test programs exercising each instruction class, validated against expected register/memory state.
  • Pair-collaborated through a shared Git-managed Vivado project, with reproducible synthesis flows.

Why It Matters

Defense and aerospace platforms increasingly run custom logic on FPGAs — radar signal processing, sensor fusion, secure compute. Knowing how to build a CPU at the RTL level is the difference between using FPGA toolchains as a black box and being able to architect application-specific pipelines.
