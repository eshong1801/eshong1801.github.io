Kinova Gen3 — Flying-Ball Catch

Programmed a Kinova Gen3 7-DOF robotic arm to catch a flying ball in real time — combining single-camera RGB trajectory estimation with Model Predictive Control for end-effector positioning.

Overview

A robotics project that pushes a commercial 7-DOF arm into a hard real-time regime: estimate where a thrown ball will be in <1 second, plan a joint trajectory to put the end-effector there, and execute it before the ball passes by.

The constraint of a single RGB camera (no depth sensor, no stereo) forced reliance on physical priors — projectile motion + known ball size — to recover the missing depth dimension.

What I Built

  • Vision pipeline detecting the ball each frame and estimating its 3D trajectory from monocular RGB.
  • Trajectory prediction combining projectile-motion physics with online state estimation.
  • Model Predictive Control (MPC) loop computing joint commands to move the end-effector to the predicted intercept point with bounded jerk and joint-limit constraints.
  • Integration with the Kinova Gen3 API for low-latency arm control.
  • End-to-end validation: real flying balls caught under lab conditions.
